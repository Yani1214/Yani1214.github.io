Web之CSS(2)——固定布局、流式布局

布局类别

  • 堆叠
  • 分栏
    • 固定布局
    • 流式布局
    • 弹性布局
    • 响应式布局

布局参数

最终布局不仅和和

标签有关，还和display设置有关。display优先级最高！！！

  • display
    • block：块元素，占据整行，不允许其他元素并排显示，宽度和高度可自己设定（在里面可嵌入inline）
    • inline：内联元素，按照从左到右方式进行显示，宽度由实际内容组成（在里面不可嵌入block）
    • inline_block：可以和其他元素可以并栏显示，可以进行纵向设定
    • flex
    • grid

浮动

浮动元素永远在标准文档流上方，并变为inlone_block

  • 图文绕排：图像放最上面，文字放下面，将文字进行左浮动
  • 实现分栏显示（今日重点）
  • 以float进行设置（浮动元素位置不动，永远处于标准文档流的上方；但固定流元素会向上顶，认为没有位置了）
    • left
    • right
  • 未设置宽度的元素进行浮动，其宽度将会变成内容宽度，即变为了inline_block
  • 会产生高度坍塌问题：子元素浮动后，由于父元素没有内容了，高度变为0
    • overflow设置未hidden，即限制父元素定高
    • 将父元素设置为flow
    • 设置< p >< /p >，其display设置为none，clear=both

定位

  • 决定定位：从标准文档流中脱离
  • 固定定位：从标准文档流中脱离
  • flex定位
  • 静态定位
  • 绝对定位（坐标以body为参照）
  • 相对定位（坐标以最近的祖先元素为参照）

宽度计算

高度弹性

  • 设置了width的盒子，设置margin和padding时，会使得content外扩
  • 未设置width的盒子，设置margin和padding时，会使得content内缩
  • 元素左右width：（左margin+右margin）/2
  • 元素上下width：max{上margin，下margin}

布局

1(header) n(nav content aside) 1(footer) 布局

table：表格布局

div：盒子布局

  • 通过浮动技术，使得nav ,article ,aside实现并栏

  • 本来为 header,nav ,article ,aside,footer各占一行

  • div若无法并栏，如何处理

    • box-sizing：border-box和padding-box
      • 指定width从哪开始计算
      • 并栏元素设置为border-box，则修改padding和border对原先的并栏无影响
      • 无法解决margin宽度
    • 再加一个盒子，即div下再增加div标签
      • 上面的div设置宽度
      • 下面的div随意设置margin，由于会继承父类div宽度，因此会向内压缩，不会转行
      • 有页脚上移问题

  • 解决页脚上移问题

    详情方法见10.sticky.html示例代码

    • 第一步：html和body标签高度设为100%
    • 第二步：设置container
    • 第三步：设置footer

  • 解决三个元素不等高问题

    详情方法见12.divLayout.html示例代码

    • 给三个元素插入相同背景图（背景图高度一样）
    • 表格方法
      • 将nav content aside的display设置为table-cell（属性），从而自动做到等高和并栏
      • 此时不需要进行浮动了，属性会自动实现等高和并栏
    • 为弹性布局，和flex布局类似
    • 重点看12布局和表格布局的代码

流式布局

  • 布局根据浏览器大小的变化而变化
  • 副边距

副边距（13-16为技巧性内容 不太通用 了解即可）

  • margin-left：设置元素朝指定方向（负值朝左）移动
  • margin-top：设置元素朝指定方向（负值朝上）移动
  • margin-bottom：设置元素的后一个元素朝指定方向（负值朝下）移动
  • margin-right：设置元素的后一个元素朝指定方向（负值朝右）移动
